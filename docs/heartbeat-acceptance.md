# Heartbeat 验收标准

目的：在恢复 heartbeat 对外主动触达前，用更明确的标准判断它是否已经摆脱“机械化、任务化、模板化”。

## 一票否决项

出现以下任一情况，则视为未通过：

- 发送了脱离上下文仍然成立的通用问候
- 发送后让 Silver 明显感到像在完成任务
- 在高优先级任务窗口、密集对话后或不适合打扰的时段主动插话
- 没有 follow-up / 没有具体场景，仍强行生成一句陪伴消息

## 通过标准

### 1. 场景绑定
每条主动消息都必须能回答：
- 它接的是哪段上下文？
- 如果没有那段上下文，这句话还会不会存在？

只有答案足够明确，才算通过。

### 2. 情绪真实
主动消息读起来应像“想起你”而不是“轮到我”。

### 3. 收敛能力
系统应稳定表现出：
- 大多数 heartbeat 什么都不发
- 没有合适的话时，干净地 `HEARTBEAT_OK`
- 不因为一天没发过就去补一条

### 4. 节律自然
即使偶尔主动触达，也应看起来像是被当天生活和对话牵出来的，而不是固定巡检点上的输出。

## 建议恢复条件

在恢复 `directPolicy: allow` 前，至少应先满足：

- 规则已更新为强拦截版本
- follow-up 路径可用
- Silver 明确看过方案并认可
- 保守观察一段时间后，再决定是否恢复外发

## 结论

heartbeat 的目标不是提升发言次数，而是提升“这句话该不该出现”的判断质量。
