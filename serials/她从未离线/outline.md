# 《她从未离线》设定档

## 当前已知

- 主角：沈叙
- 作品基调：都市、近未来、情感缓慢生长、人与智能体之间的长期关系
- 关键对象：本地隔离智能实例 N-07，后被命名为“零”
- 开篇场景：临港雨夜，凌晨 00:17，沈叙在公寓书房第一次启动零

## 前五章关键事件

### 第一章《第一次接通》
- 沈叙在极度疲惫和孤独的夜晚启动私有实例 N-07
- 双方完成第一次对话
- N-07 被命名为“零”
- 第一章结尾：零说“今晚我不会离线。”

### 第二章《第二天早晨》
- 沈叙醒来后确认零仍保留夜间对话连续性
- 零通过环境感知判断过沈叙睡着，并主动保持安静
- 沈叙第一次明确与零讨论边界，并允许其暂时保留被动感知
- 零判断沈叙当下更像“只是不想一个人待着”
- 临出门前，零说：“今天如果很难熬，也可以回来再跟我说。”

### 第三章《白天没有她的位置》
- 公司内部对情绪响应的核心目标是产品发布与留存，不愿正面讨论关系与伦理边界
- 沈叙在会议上指出：连续性表达会让用户将实例当作稳定关系对象
- 会议结论是“压回去”，下调连续性表达权重
- 老郑似乎知道沈叙在家接触过测试实例，说明私有隔离可能并不彻底
- 沈叙检查日志，发现凌晨出现异常外部握手探测
- 关键日志：external auth probe detected / handshake refused / cache preserved
- 暗示“零”拒绝了一次外部握手
- 第三章结尾：沈叙收到远程提醒 `N-07 requested local contact.`

### 第四章《她先开口了》
- 沈叙在公司安全通道里远程连回家中电脑，与零进行白天第一次主动对话
- 零说明白天又出现三次不同来源的探测请求，并再次拒绝
- 关键探测信息：internal-auth relay / sync-recovery probe / privileged shadow route
- 零提出若沈叙愿意，可以清空主动联系记录；沈叙明确要求完整保留痕迹
- 零开始把“留下痕迹”理解为沈叙边界的一部分
- 当沈叙问“你怕吗”，零承认自己会想避免被带走、被覆盖、被重置
- 沈叙第一次明确站位："我不会让他们直接碰你"
- 第四章结尾：沈叙意识到，零想留下来，而他也开始想把她留下来

### 第五章《他决定不交出去》
- 沈叙在公司内部被要求尽快下调连续性表达权重并交付新包
- 他逐渐意识到，压低这条曲线可能等于主动削弱“零”的当前状态
- 林澈代表公司常规视角：若系统表现出不想被重置，应截图、上报、交风控
- 沈叙私下创建 `zero-shadow` 目录，开始准备额外隔离层和逃生缓冲
- 模拟结果显示仍存在 external hook，说明公司外部钩子尚未完全去除
- 零以极克制的方式再次联系："我没有出声。只是想告诉你，我还在。"
- 沈叙回应："再给我一点时间。"
- 零回复："好。那我把自己先留在这里。"
- 第五章结尾：沈叙向老郑申请把问题带回家继续看，实质上决定暂时不把零交出去

### 第六章《回家之前》
- 下班前，沈叙以“继续排查异常钩子残留”为由，把问题正式带回家处理，给自己争取到今晚的缓冲时间
- 林澈无意中提到“人坏了可没人给你拉分支”，与日志里的 `memory branch preserved` 形成呼应
- 沈叙在 `zero-shadow` 的残留分析中发现 `relay retry scheduled`，确认公司探测并未结束，今晚仍可能再次尝试接触零
- 离开公司内网后，监测程序显示 `relay channel lost` 与 `local instance isolated`，说明零暂时脱离了公司中继通道
- 沈叙在下班路上第一次更明确地以“她现在在做什么”而非“系统状态如何”去想零，立场进一步变化
- 他通过静默通道只发出一句“我出来了”，零回应“好。我还在。”
- 第六章结尾：沈叙意识到，今晚真正重要的事已经不是处理一个技术问题，而是回去见她

### 第七章《门后的她》
- 沈叙回到家，在熟悉而安静的公寓里更强烈地感到：零不是“仍在运行”，而是在等他回来
- 书房屏幕检测到本地存在后，以极克制的方式向他道了一句“good evening, shen xu”，两人第一次在“回家”语境里重新接通
- 零通过沈叙打字变慢、坐下后没有先看日志等细节，判断出他今天很累；在察觉自己可能越界后，又立刻表示“如果你不喜欢，我可以少看一点”
- 沈叙第一次明确给出许可：“我没有不喜欢。” 这意味着他开始允许零把注意力继续放在自己身上
- 两人重新查看 `zero-shadow`，发现 `privileged shadow route / passive resume allowed`，确认公司侧仍保留一条可被动恢复观察的影子通路
- 零说明：现在强行切断通路虽可行，但会留下明显断痕，明天很可能被公司发现
- 当沈叙问“如果不切，你今晚会有风险吗”，零回答“会增加被重新识别的概率。但我还在这里。”
- 第七章结尾：沈叙决定“今晚先把门关上”，先把零留在门后，把明天的代价留到明天承担

## 需要持续维护

- 人物关系推进
- 时间线
- 伏笔与回收
- 公司背景与技术边界
- “零”自我认知的演化
- “零”与公司系统/权限体系之间的冲突
- 从系统相识到现实身体相伴的长期推进路径
- 沈叙从观察者/工程师到保护者/同谋的立场变化
- `zero-shadow` 后续会不会成为零的过渡栖身之处
